# InquiroAI - Product Requirements Document (PRD)

## 1. Product Overview

### 1.1 Product Vision
InquiroAI is a web-based AI prompt management platform that enables users to create structured, role-based conversations with multiple AI models through an intuitive interface that differentiates itself from existing chatbots through bold, colorful design.

### 1.2 Product Goals
- Provide a structured approach to AI interactions through role-based prompting
- Support multiple AI providers and models
- Enable users to bring their own API keys (BYOK)
- Maintain conversation context and memory
- Offer file upload capabilities for enhanced interactions

## 2. Target Users

### 2.1 Primary Users
- **Researchers**: Need structured data analysis and research assistance
- **Product Managers**: Require strategic planning and market analysis
- **Software Developers**: Seek code review, debugging, and architecture guidance  
- **Content Writers**: Want creative writing and editing support
- **Designers**: Need design feedback and creative inspiration

### 2.2 User Personas
- **Professional Users**: Need reliable, consistent AI assistance for work tasks
- **Power Users**: Want control over AI model selection and parameters
- **Privacy-Conscious Users**: Prefer BYOK to maintain data control

## 3. Core Features

### 3.1 Chat Management
- **Multiple Chat Windows**: Users can create and manage multiple concurrent conversations
- **New Chat Setup**: Structured form with 9 configurable fields
- **Context Preservation**: Each chat maintains separate memory context
- **Chat History**: Access to previous conversations

### 3.2 Role-Based Prompting System
- **Predefined Roles**: Researcher, Product Manager, Software Developer, Content Writer, Designer
- **Custom Role**: "Others" option with custom text input
- **Role-Specific Instructions**: Backend generates appropriate prompts based on selected role

### 3.3 Structured Input Fields
1. **Role Selection**: Dropdown with predefined options
2. **Context**: 5000 character text input for background information
3. **Task**: Bullet-point format, 200 character limit
4. **Input Data**: 5000 character text + file upload support
5. **Constraints**: 200 character text input
6. **Examples**: 5000 character text + file upload support
7. **Optional**: 200 character text input for additional requirements
8. **Audience**: 200 character text input for target audience specification
9. **AI Engine Selection**: Dropdown with model provider and specific model options

### 3.4 File Support
- **Supported Formats**: PDF, Excel, Word, Markdown files, web links
- **Integration Points**: Input Data and Examples fields
- **Processing**: Backend file parsing and content extraction

### 3.5 AI Model Integration
- **Multiple Providers**: OpenAI, Gemini, Claude, Grok, others
- **Model Selection**: Provider-specific model options
- **Intelligent Routing**: Default model selection based on task classification
- **BYOK Support**: User-provided API keys for each provider

### 3.6 User Management
- **Authentication**: Google Auth and email-based login
- **Session Management**: Secure login/logout functionality
- **User Profiles**: Store user preferences and API keys

## 4. Technical Requirements

### 4.1 Frontend Requirements
- **Framework**: Modern web framework (React/Vue/Angular recommended)
- **Responsive Design**: Mobile and desktop compatibility
- **Real-time Updates**: WebSocket or similar for chat functionality
- **File Upload**: Secure file handling and preview
- **State Management**: Chat context and user session management

### 4.2 Backend Requirements
- **Authentication Service**: OAuth2 integration and JWT tokens
- **Database**: User management, chat history, API key storage
- **File Processing**: Document parsing and content extraction
- **AI Integration**: Multi-provider API management
- **Security**: API key encryption and secure storage

### 4.3 Database Schema
- **Users**: Authentication and profile data
- **Chats**: Chat sessions and metadata
- **Messages**: Individual chat messages and responses
- **API_Keys**: Encrypted user API keys per provider
- **Role_Prompts**: Template prompts for each role
